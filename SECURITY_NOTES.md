# Протокол инцидентов безопасности и правила работы с секретами

## Инцидент безопасности: 17 июня 2026 г.

### Описание инцидента
Во время настройки интеграции Git-репозитория с GitHub, Personal Access Token (PAT) был передан в качестве аргумента bash-команды `git remote add origin https://<TOKEN>@github.com...`. Это привело к утечке токена в открытом текстовом формате в логах выполнения команд и истории чата ИИ-ассистента.

### Принятые меры по устранению
1. **Отзыв токена**: Скомпрометированный токен `ghp_Ps69Yj...` был немедленно отозван в панели управления GitHub.
2. **Очистка файлов**: 
   * Токен удален из локального конфигурационного файла `credentials.env`.
   * Полностью удалены файлы кэша учетных данных `~/.git-credentials` локально и на VPS (`/root/.git-credentials`), где токен хранился в открытом виде.
3. **Безопасное переназначение**: Новый токен настроен через интерактивный механизм Git (`git credential approve` через Python-скрипт) без сохранения в историю команд bash и историю логов чата.
4. **Проверка**: Проведено сканирование файлов проекта и VPS с помощью `grep`, подтвердившее отсутствие открытых следов токена.

## Правила безопасности проекта

Для предотвращения подобных утечек в будущем все разработчики и ИИ-ассистенты обязаны следовать правилам:

1. **Никаких секретов в командах**: Запрещается запускать команды терминала, содержащие реальные API-ключи, токены или пароли в аргументах (например, `echo "secret"`, `git remote add ... <token>`).
2. **Использование системных менеджеров паролей**: Для авторизации Git использовать `git credential.helper` (кэш или хранилище), передавая учетные данные через стандартный поток ввода (`stdin`) или интерактивно.
3. **Безопасность `.env`**: Файлы конфигураций `credentials.env`, `.env`, `*.env` должны быть всегда занесены в `.gitignore`. Их категорически запрещается отправлять в репозиторий.
4. **Хранение токенов**: В файле `credentials.env` вместо реальных токенов GitHub должен храниться плейсхолдер `your_new_github_token_here`. Git-авторизация должна работать на уровне операционной системы через credential-helper, а не через парсинг переменных окружения проекта.
